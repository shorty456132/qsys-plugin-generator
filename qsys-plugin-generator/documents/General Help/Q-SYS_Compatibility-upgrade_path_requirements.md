# Upgrade Requirements

Note these requirements when upgrading the Q-SYS Core processor and Q-SYS peripherals.

[Upgrading from Version 10 Releases](javascript:void(0))

If you are upgrading from 10.0.x or 10.1.x, no additional upgrade steps are necessary.

Upgrade directly to 10.2.x.

[Upgrading from Version 9 Releases](javascript:void(0))

If you are upgrading from version 9.13.x, no additional steps are necessary.

If you are upgrading from version 9.1.x through 9.12.x:

1. Upgrade to version 9.13.x.
2. Upgrade to version 10.1.x.

If you are upgrading from version 9.0.x:

1. Upgrade to version 9.1.x.
2. Upgrade to version 9.13.x.
3. Upgrade to version 10.1.x.

[Upgrading from Version 8 Releases](javascript:void(0))

If you are upgrading from any Version 8 release:

1. Upgrade to version 9.1.x.
2. Upgrade to version 9.13.x.
3. Upgrade to version 10.1.x.

[Upgrading from Version 7 Releases](javascript:void(0))

If you are upgrading from version 7.2.1:

1. Upgrade to version 9.1.x.
2. Upgrade to version 9.13.x.
3. Upgrade to version 10.1.x.

If you are upgrading from version 7.2.0, 7.1.x, or 7.0.x:

1. Upgrade to version 7.2.1.
2. Upgrade to version 9.1.x.
3. Upgrade to version 9.13.x.
4. Upgrade to version 10.1.x.

[Upgrading from Version 6 Releases](javascript:void(0))

If you are upgrading from version 6.2.x or 6.1.x:

1. Upgrade to version 7.2.1.
2. Save the design in QDS version 9.0.0 to upgrade the file format. (There is no need to upgrade the Core and peripherals to 9.0.0.)
3. Upgrade to version 9.1.x.
4. Upgrade to version 9.13.x.
5. Upgrade to version 10.1.x.

[Upgrading from Version 5 Releases](javascript:void(0))

If you are upgrading from version 5.4.x, 5.3.x, 5.2.x, 5.1.x, or 5.0.x:

1. Upgrade to version 6.0.0. See the version 6.0.0 [Release Notes](https://www.qsys.com/resources/software-and-firmware/q-sys-designer/archives/) for upgrade requirements for that version. (There is no need to update peripherals to version 6.0.0 in advance of updating to 7.2.1. Once the Q-SYS Core is running 7.2.1, it will update connected peripherals running version 5.x to version 7.2.1.)
2. Upgrade to version 7.2.1.
3. Save the design in QDS version 9.0.0 to upgrade the file format. (There is no need to upgrade the Core and peripherals to 9.0.0.)
4. Upgrade to version 9.1.x.
5. Upgrade to version 9.13.x.
6. Upgrade to version 10.1.x.

[Upgrading from Version 4 and Earlier Releases](javascript:void(0))

If you are upgrading from a version earlier than 4.2:

1. Upgrade to version 4.2. See the version 4.2 [Release Notes](https://www.qsys.com/resources/software-and-firmware/q-sys-designer/archives/) for upgrade requirements for that version.
2. Upgrade to version 6.0.0. See the version 6.0.0 [Release Notes](https://www.qsys.com/resources/software-and-firmware/q-sys-designer/archives/) for upgrade requirements for that version.
3. Upgrade to version 7.2.1.
4. Save the design in QDS version 9.0.0 to upgrade the file format. (There is no need to upgrade the Core and peripherals to 9.0.0.)
5. Upgrade to version 9.1.x.
6. Upgrade to version 9.13.x.
7. Upgrade to version 10.1.x.

**CAUTION:** Designs you create or update with a newer QDS version and then open in an older version are likely to be incompatible and unusable with the older version of software. Before upgrading, create backup copies of your Q-SYS design files for both the current version and older versions, and back up all media on your Q-SYS Core processor. To avoid overwriting design file backups, do not open backup files with newer versions of QDS.
